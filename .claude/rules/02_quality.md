# Контроль качества — Yummy

## Тесты

- Новая функция/эндпоинт = тест. Без теста фича не считается готовой.
- Запуск: `rm -f spasibox.db* && .venv/bin/python -m pytest -q` (текущий baseline: 81).
- Изоляция обязательна: API-тесты монкейпатчат `app.main.store` на свежий
  `Store(path=tmp_path/...)` — иначе процессный синглтон истощает инвентарь между
  тестами. `tests/conftest.py` глушит `TELEGRAM_BOT_TOKEN`/`ANTHROPIC_API_KEY` ДО
  импорта app (TestClient выполняет BackgroundTasks синхронно → иначе реальные
  Telegram-сообщения из тестов).
- Именование: `test_<поведение>_<условие>` (напр. `test_cancel_after_window_start_blocked`).
- Структура AAA (Arrange–Act–Assert).
- Async-функции тестировать через `asyncio.run(...)` в обычных `def` (pytest-asyncio
  не зависимость проекта).

## Типы

- Аннотации типов на всех сигнатурах (PEP 8 + type hints).
- Валидация на границе: Pydantic на каждом входе в API, `Field(...)` вместо
  ручных проверок где выразимо.

## Безопасность (проверять при каждом изменении auth/данных)

- Секретов в коде нет (`grep` по токен-паттернам). Всё через env/`.env`.
- Параметризованный SQL (плейсхолдеры), не конкатенация.
- Владение ресурсом — через запрос с `WHERE user_id=?`, не через ID из тела.
- Ответы не содержат pw_hash/токенов.
- Rate-limit на write-эндпоинтах и AI-эндпоинтах (отдельные лимитеры).

## Линт/формат (опционально — не установлены в venv)

- Если появятся: `ruff check app tests tools` + `black app tests tools`.
- Формат-хук в `.claude/settings.json` самоохранный: молчит, если ruff/black нет,
  но всегда прогоняет `py_compile` (ловит синтаксис).
