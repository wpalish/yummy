# TDD Workflow — RED → GREEN → REFACTOR

Для каждого изменения строго по циклу. Стек — pytest, НЕ vitest.

## RED
1. Написать ПАДАЮЩИЙ тест первым в `tests/test_<area>.py`.
2. Запустить: `rm -f spasibox.db* && .venv/bin/python -m pytest tests/test_<area>.py -q`
   — убедиться, что ПАДАЕТ.
3. Тест проверяет ОДНО поведение.

## GREEN
4. Написать МИНИМАЛЬНЫЙ код, чтобы тест прошёл.
5. Без лишнего, без преждевременной оптимизации.
6. Запустить — ПРОХОДИТ.

## REFACTOR
7. Убрать дублирование, улучшить именование.
8. Прогнать ВСЕ тесты: `.venv/bin/python -m pytest -q` — ничего не сломалось.
9. Повторить для следующего поведения.

## Правила проекта

- Имена: `test_<поведение>_<условие>`.
- Изоляция: API-тесты монкейпатчат `app.main.store` на свежий `Store(path=tmp_path/...)`.
- Async — через `asyncio.run(...)` в обычной `def` (pytest-asyncio не подключён).
- Структура AAA. Паттерны брать из существующих тестов (`test_flow.py`, `test_accounts.py`).
- НИКОГДА не писать прод-код без падающего теста.
